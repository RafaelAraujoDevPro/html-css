<Tags de importância Semântica HTML>

Tag <header>

O elemento HTML <header> representa um grupo de suporte introdutório ou navegacional. Pode conter alguns elementos de cabeçalho mas também outros elementos como em logo, seções de cabeçalho, formulário de pesquisa, e outros.

O elemento <header> não é separador de conteúdo (sextioning content), portanto,não introduz uma nova seção no out line.

  • Tag usada para definir o cabeçalho do página. Contém semântica que facilita para o SEO da página.
  • Esta tag pode difinir um cabeçalho relativo ao seu contexto.
  • Normalmente esta tag possui tags de titulos h1, h2, h3, h4, h5 e h6.
  • Pode ser colocado um logotipo, ícone ou imagem em um<header>.

    <header> não possui atributo próprio e aceita todos os atributos globais.

Exemplo:

                    
                        <header>
                            <h1>Título da Página Principal</h1>
                            <img src="mdn-logo-sm.png" alt="MDN logo">
                        </header>

Tag <main>

A tag <main> é entendida como conteúdo pricipal do documento HTML, e está diretamente relacionado com o tópico central da página. Semânticamente tem função central da página aplicada no SEO. O mesmo deve ser único na página, ou seja, o conteúdo dentro da tag <main> não pode se repetir em outras tags HTML.

A tag <main> não pode ser filho dos elementos <article>, <aside>, <footer>, <header> ou <nav>. Autores não devem incluir mais de um elemento <main> no mesmo documento.

Não possui atributo próprio. Aceita todos os atributos globais.

Exemplo:

                    
                        <main>
                        <h1>Maçãs</h1>
                        <p>A maçã é a fruta pomácea da macieira.</p>
                        <article>
                        <h2>Vermelho delicioso</h2>
                        <p>Estas maçãs vermelhas brilhantes são as mais comumente encontradas em muitos supermercados.</p>
                        <p>... </p>
                        <p>... </p>
                        </article>
                        <article>
                        <h2>Granny Smith</h2>
                        <p>Essas suculentas maçãs verdes, são um ótimo recheio para
                        torta de maçã.</p>
                        <p>... </p>
                        <p>... </p>
                        </article>
                        </main>

Tag <section>

O elemento HTML <section> representa uma seção genérica contida em um documento HTML, geralmente com um título, quando não existir um elemento semântico mais específico para representá-lo.

Por exemplo, um menu de navegação deve estar dentro de um elemento <nav>, mas uma lista de resultados de pesquisa ou a exibição de um mapa e seus controles não possuem elementos específicos, e podem ser colocados dentro de uma tag <section>

Não possui atributo próprio. Aceita todos os atributos globais.

  • Cada <section> seve ser identificado, geralmente incluindo um cabeçalho (elemento <h1> até <h6>) como um filho do elemento <section>.
  • Se faz sentido distribuir separadamente o conte conteúdo de um elemento <section>, use um elemento <article> em seu lugar.
  • Não use o elemento <section> como um container genérico; para isso existe a tag <div>, especialmente quando a seção é apenas com propósito de estilização. uma regra de ouro é "usar quando uma seção deve aparecer logicamente na estrutura de um documento"

Exemplo:

                    
                        <section>
                        <h2>Conteúdo sem tag específica</h2>
                        <p>Este conteúdo não se encaixa em uma tag específica semânticamente,
                        então empregamos a tag <section> para agrupar seus elementos</p>
                        <section>

Tag <article>

O elemento HTML <article> representa uma composição independente em um documento, página, aplicação ou site, ou o que seja destinado a ser distribuído de forma independente ou reutilizável. Este pode ser o post de um fórum, um artigo de revista ou jornal e etc...

Além do seu conteúdo principal o elemento <article> tipicamente contém um cabeçalho dentro de uma tag <header> e algumas vezes um rodapé. Para indentificar se o conteúdo é independente do tópico principal temos que perceber de o mesmo sentido isoladamente em ralação ao conteúdo principal.

  • Quando um elemento <article> está aninhado, o elemento interior representa um artigo relacionado com o elemento exterior. Por exemplo, os comentários do post de um blog podem ser elementos <article> aninhados a outro <article> representando o post do blog.
  • Informações sobre o autor de um elemento <article> podem ser fornecidas através do elemento <address>, mas não se aplicam aos elementos <article> aninhados.
  • A data e hora de publicação de um elemento <article> pode ser descrita usando o atributo pubdate de um elemento <time>

Não possui atributo próprio. Aceita todos os atributos globais.

Exemplo:

                    
                            <article>
                            <h2>Conteúdo independente</h2>
                            <p>Este conteúdo é independente em relação ao conteúdo
                            principal, então devemos usar a tag semântica <article>,
                            para que os navegadores possam indentificá-lo como 
                            artigo.</p>
                            </article>
