O que é FlexBox?

O FlexBox tem como meta ser um modo mais eficiente para criar leiautes, alinhar e distribuir de maneiras avançadas os espaços entre os elementos de uma interface, mesmo quando as dimensções destes itens são desconhecidos e/ou dinâmicas (dai o termo flex).

FlexBox é um módulo completo de propriedades com a finalidade de posicionar os elementos filhos em seus respectivos elementos pais para uma boa compatibilidade entre o browsers.

Para aplicar FlexBox devemos declarar suas propriedades tanto no, container, (elemento pai que chamamos de fex container)quanto nos elementos filhos que chamamos de (flex itens).

Elementos

Se o laiaute "padrão" é baseado na direções block e inline, o leiaute Flex é baseado em direções "flex flow". Veja o diagrama da especificação, explicando a ideia central por trás do leiaute Flex.

Ao se utilizar o flexbox, é preciso saber que todas as operações realizada relacionam-se a dois eixos: o eixo principal e o eixo transversal.

O eixo principal é difinido através da propriedade flex-direction e o eixo transversal encontra-se na direção perpendicular a ele

  • Eixo principal → o eixo principal de um flex container definido, será o eixo primário e ao longo dele serão inseridos os flex items.
  • main-start/main-end → os flex items são insetidos dentro do container começando pelo lado start, indo em direção ao lado end.
  • Tamanho principal → A largura ou altura de um flex item, dependendo da direção do container, é o tamanho principal do item.A propriedade de tamanho principal de um flex item deve ser tanto width quato height, dependendo de qual delas estiver na direção principal.
  • Eixo transversal → O eixo perpendicular ao eixo principal é chamado de eixo transversal, sua direção depende da direção do eixo principal.
  • cross-start/cross-end → Linhas flex são preenchidas com itens e adicionadas aou container, começando pelo lado cross start do flex em direção ao lado cross end.
  • cross size → A largura ou autura de um flex item, dependendo do que estiver na dimensão transversal, é o cross size do ítem, o que estiver na transversal.

flex container

A área de um documento que faz uso do flexbox é chamada de flex container. Para criar essa estrutura é preciso declarar a propriedade display do elemento representado pelo container com os valores flex ou inline-flex Desse modo, os elementos-filhos desse contêiner se tornarm do tipo flex. Como todas as propriedades no CSS possuem valores padrão, ao criar um contêiner flex, os elementos nele contidos apresentarão o seguinte valores:

Se houver mais ítens do que é possível caber no contêiner, não haverá uma quebra de linha ; ao invés disso, irão ultrapassar o limite horizontal da página. Se alguns elementos forem mais alto que os outros , todos os itens se estenderão ao longo do eixo transversal para preencher seu tamanho total.

Propriedade flex-direction

A propriedade flex-direction permite alterar a direção na qual os elementos flex serão exibidos ao longo do eixo principal. Definindo a propriedade flex-direction como row-reverse (linha reversa) ainda teremos os elementos dispostos em uma linha, entretanto, as linhas inicial e final serão trocadas (da direita para esquerda).

Se mudarmos a flex-direction para column (coluna), o eixo principal exibirá os elementos em uma coluna. trocando o valor para column-reverse (coluna reversa) fará vom que as linhas inicial e final sejam novamente trocadas (de baixo para cima).

Exemplo de sintaxe

                    
                        .boxdirection{
                            display: flex;
                            flex-direction: row-reverse;
                        }
                    
                
                    
                        <div class="boxdirection">
                            <div>item 1</div>
                            <div>etem 2</div>
                            <div>etem 3</div>
                        </div>

  • now-reverse: exibição em linha da direita pra esquerda;
  • column: exibição em coluna de cima pra baixo;

Flex-wrap

Embora o flexbox seja um modelo unidimensional, é possível fazer com que elementos flexsejam agrupados em múltiplas linhas. Ao fazer isso, considera-se cada linha como um novo contêiner Flex. Qualquer distribuição espacial ocorrerá ao longo dessa linha, sem referência às linhas de ambos os lados. Para gerar a quebra automática das linhas adicione a propriedade flex-wrap com o valor wrap. Assim, de elementos forem muito grandes para serem exibidos em uma única linha, eles serão agrupados em outras linhas.

O exemplo prático abaixo contém elementos flex aos quais foi dada uma determinada largura, cuja soma totoliza um valor grande demais para caber no contêiner. Visto que a propriedade flex-wrap está definida como wrap, os itens serão reorganizados em mais de uma linha.

                    
                        .boxwrap {
                            display: flex;
                            flex-wrap: wrap;
                        }
                    
                
                    
                        <div class="boxwrap">
                            <div>item 1</div>
                            <div>etem 2</div>
                            <div>etem 3</div>
                        </div>

Expansão, encolhimento e tamanho dos elementos flex

Para ter mais controle sobra os elementos flex é possível alterá-los diretamente utilizando as três propriedades abaixo:

  • flex-grow
  • flex-shrink
  • flex-basis

Quando se modifica o valor das propriedades , altera-se a forma que o espaço disponível é distribuido entre os elementos. Tal conceito de espaço disponível também é relevante quando se trata do alinhamento

As propriedades a seguir tratam dessa distribuição do espaço disponível do elemento pai entre seus elementos filhos, cada uma da sua maneira; seja todos os elementos crescendo proporcionalmente ou não, seja preenchendo espaço disponível, ou ainda distribuição dos espaços vazios entre eles

flex-basis

Flex-basis serve para especificar o tamanho inicial dos "flex-items" antes de um eventual espaço livre ou faltante do flex-conteiner ser dividido e distribuido, segundo o que constar em flex-grow e flex-shrink. Em outras palavras, flex-basis é a melhor maneira de definir tamanho de "flex-items", ao invés de usar "width" ou "height", pois ele os manipula de acordo com a orinteção do "flex-direction".

A propriedade flex-basis é do tipo flex-items, deve ser aplicada nos elementos filhos. flex-basis se aplica ao tamanho de todos elementos filhos ao mesmo tempo. Ela segue a direção do main-axis, portanto se mudamos o flex-direction ela mudará o eixo á ser manipulado.

  • flex-direction: row → manipulará a largura (width);
  • flex-direction: column → manipulará a altura (height).
flex-basis está como padrão com o valor auto, quando o seu elemento pai está como display-flex. Isso faz com que o elemento tenha o tamanho do seu conteúdo. Quando adicionamos um valor em pixels, todos os seus elementos filhos terão essa medida por igual independente do tamanho de seus conteúdos.

Propriedade flex-grow

Flex-grow define como os elementos flex-items de comportam caso haja espaço disponível no flex-container ("grow" significa crescer). Com a propriedade flex-grow definida com um número inteiro positivo, os elementos flex podem crescer ao longo do eixo principal, recebendo uma cota igual de espaço live do conteiner a partir do valor mínimo estabelecido no flex-basis. Isto fará com que o elemento se estique e ocupe qualquer espaço disponível nesse eixo ou uma proporção dele, caso outros elementos-irmãos também possam crescer.

Flex-grow é do tipo flex-itens então devemos aplica-la nos elementos filhos. Um número inteiro positivo como valor para ela fará com que os elementos cresçam de maneira fracionada de acordo com o respectivo valor. Vamos ver um exemplo:

Propriedade flex-shrink

(Shrink significa "encolher"), logo, a propriedade flex-shrink é a que faz os elementos filhos se ajustarem, redimensionando o tamanho para diminuirem quando o espaço do conteiner diminui, distribuindo cotas de espaços livres. Esse redimensionamento se limita até o tamanho do conteúdo contido nele mais o tamanho do padding caso haja algum. O seu valor padrão é o número inteiro positivo "1".

flex-shrink é também do tipo flex-itens, sendo assim aplica-se nos elementos filhos. Se aplicarmos o valor "0" o elemento se posiciona fixo e não diminúi seguindo o layout. Avreviatura para propriedades flex

As propriedades flex-grow, flex-shrink e flex-basis raramente são empregadas de maneira individual. Usualmente, elas são combinadas através da propriedade de abreviação flex. A abreviatura flex permite definir os três valores na seguinte ordem: flex-grow, flex-shrink, flex-basis.

Vamos ver o exemplo de sintaxe abaixo

Há ainda alguns valores de abreviação predefinidos, que cobrem a maioria dos casos de uso. São aplicados normalmente para suprir as necessidades práticas da estilização flexbox.

  • flex: initial; → reseta os elementos para valores padrões do FlexBox, sendo equivalente a flex: 1 1 auto;.
  • flex: auto; → é o equivalente a flex: 1 1 0; que fará os elementos preencher o contêiner ou diminuir se necessário, pra evitar o transbordamento lateral do layout.
  • flex: none; → irá criar um elemento flex totalmente inflexível, sendo equivalente à felx: 0 0 auto;.
  • flex: número inteiro; → equivalente à flex: 3; 4, 2 e etc, onde os elementos podem crescer ou diminuir a partir da propriedade flex: basis; com o valor nulo.

Alinhamento e justificação

Um atributo chave do flexbox é a capacidade de alinhar e justificar os elementos flex nos eixos principal e transversal e distribuir o espaço livre entre eles.

Propriedade align-items

A propriedade align-items tem a função de alinhar os elementos no eixo transversal (cross-axis). align-items é aplicada no elemento pai para alinhar os elementos filhos no espaço do seu contêiner. O valor inicial padrão desta propriedade é stretch que faz os elementos flex se estenderem até a maior altura entre eles.

Para que align-items faça sua função o elemento pai deve está com o display: flex declarado, assim como também é necessário para as outras propriedades do tipo flex. Os valores que se aplicam nesta propriedade são os seguintes: stretch, flex-start, flex-end e center.

  • stretch → esta propriedade faz os elementos flex se estenderem até a maior altura entre eles.
  • flex-start → alinha os elementos no início do eixo transversal (cross-axis), no topo do contêiner.
  • flex-end → alinha os elementos no fim do eixo transversal (cross-axis), na base do contêiner.
  • center → alinha os elementos no centro do eixo transversal (cross-axis).

Propriedade justify-content

A propriedade justify-content tem a função de alinhar os elementos no eixo principal (main-axis) seguindo a direção definida pela propriedade flex-direction. justify-content deve ser declarada no elemento pai para alinhar os elementos filhos. Para cumprir sua função o elemento pai também precisa estar com display-flex declarado e também ter uma largura definida.

O valor padrão inicial para justify-content é flex-start, que alinha os elementos no início do eixo principal. Os valores para esta propriedade sao os seguinte: flex-start, flex-end, center, space-around, space-between e space-evenly.

  • flex-start → alinha os elementos para o início do eixo principal
  • flex-end → alinha os elementos para o fim do eixo principal.
  • center → alinha os elementos para o centro do eixo principal.
  • space-around → alinha os elementos distribuindo o espaço vazio entre eles, com um espaço maior entre eles e um espaço menor nas laterais.
  • apace-between → alinha os elementos distribuindo o espaço vazio entre eles, onde o espaço entre eles são iguais, mas, sem espaço nas laterais.
  • space-evenly → alinha os elementos distribuindo o espaço vazio entre eles, distribuindo espaço vazio com uma única medida entre os elementos as e também nas laterais.

Align-self

A propriedade align-self tem a função de alinhar um elemento alvo específico dentro de um contêiner flex. O align-self alinha o elemento no mesmo eixo do align-items (eixo transversal, "cross-axeis"). Quando aplicado ele sobreescreve o valor de align-items O seu valor padrão é o auto, que faz o elemento alvo herdar o valor de align-items de seu elemento pai. Todos os valores de align-items também estão disponíveis para o align-self.

  • stretch → esta propriedade faz os elementos flex se estenderem até a maior altura entre eles.
  • flex-start → alinha os elementos no início do eixo transversal (cross-axis), no topo do contêiner.
  • flex-end → alinha os elementos no fim do eixo transversal (cross-axis), na base do contêiner.
  • center → alinha os elementos no centro do eixo transversal (cross-axis).

O valor da palavra-chave normal tem efeitos diferente dependendo do modo de layout em que o elemento se encontra:

  • Em layouts pisition: absolute o valor normal se comporta como o valor start em caixas absolutamente aposicionadas replaced, e como stretch em caixa absolutamente posicionadas all other.
  • Em layouts position: static absolutamente posicionados, a palavra-chave se comporta como stretch.
  • Para itens-flex, a palavra-chave se comporta como stretch
  • Para itens-grid, a palavra-chave se comporta similar ao de stretch, com exceção de caixas com aspect ratio ou tamanhos intrencicos onde se comportam como start.
  • A propriedade não se aplica para caixas block-level.

Propriedade Order

A propriedade order nos permite alterar a ordem original em que os elementos são apresentados, sem alterar a ordem que estão no HTML. Essa propriedade é projetada para colocar os itens em grupos ordinais. O que isso significa é que os itens recebem um número inteiro que representa seu grupo. Os itens são colocados na ordem visual de acordo com esse número inteiro, os valores mais baixos primeiro. Se mais de um item tiver o mesmo nalor inteiro, dentro desse grupo os itens serão dispostos de acordo com a ordem de origem. O valor padrão de order é "0".

Como exemplo, temos 5 itens flexíveis e vamos atribuir order com os valores da seguinte forma

  • Item de origem 1 → order: 2
  • Item de origem 2 → order: 3
  • Item de origem 3 → order: 1
  • Item de origem 4 → order: 3
  • Item de origem 5 → order: 1

Estes itens seriam exibidos na página na seguinte ordem:

  • Item de origem 3 → order: 1
  • Item de origem 5 → order: 1
  • Item de origem 1 → order: 2
  • Item de origem 2 → order: 3
  • Item de origem 4 → order: 3

Anotações - Observações

- Para um layout com uma responsividade mais eficaz o ideal é declará como flex, cada elemento, um dentro do outro simultâneos.
- a propriedade flex-basis quando declarada com o valor um de número de pixels soma este valor a partir do contúdo do elemento.
- Quando se trata de altura no FlexBox percebemos que os valores do elemento pai e do elemento filho se na estiverem declarado como um valor fixo, devem alternar entre si um valor fixo ("pixels" e "em" ) e um valor relativo (% porcentagem). Não podendo ser dois valores de porcentagem para eles.
