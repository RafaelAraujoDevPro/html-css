Conceitos básicos de Grid Layout

CSS Grid Layout flex_wrap introduz um sistema bi-dimensional de grid (literalmente "grades") para CSS. Grids podem ser usados para o design de layouts de grandes seções de uma webpage, assim como de pequenos elementos de interface. Esse artigo apresenta o CSS Grid Layout Level 1. As funcionalidades demonstradas neste resumo serão posteriormente explicadas em maiores detalhes nas demais seções desse guia.

O que é Grid Layout? flex_wrap

Grid é uma malha formada pela interseção de um conjunto de linhas horizontais e um conjunto de linhas verticais. Um dos conjuntos define colunas e outro define linhas. Dentro de um grid, respeitando-se a configuração criada pelas suas linhas, pode-se inserir elementos da marcação. As CSS grid layout preveem as seguintes funcionalidades:

Dimensões fixas ou flexíveis

Podemos criar grids com dimensões fixas ou flexíveis que vai depender do valor declados para as propriedades de grid. As dimensões fixas são decladas com pixels, já as dimensões flexíveis são declaradas com porcentagem (%) ou fraçoes (fr).

Criação de grids adicionais

Além da possibilidade de se criar um grid inicial para o layout a especificação prevê também a possibilidade de se adicionar linhas e colunas para layoutar conteúdos adicionados fora do grid inicial. Com isso podemos adicionar tantas colunas quantas forem necessárias em um grid contêiner existente.

Alinhamento e Justificação

Assim como no flexbox, no grid layout também podemos alinhar os elementos itens com as mesmas propriedades de justificação e alinhamento. As propriedades e valores são as mesmas declaradas para o flexbox.

Controle sobre conteúdos sobrepostos

Em uma célula do grid podem ser posicionados mais de um item da página e também é possivel que se defina sobreposição parcial de áreas. Esse controle de camadas é feito com uso de z-index.

CSS Grid Layout é um poderosa específicação que se for combinada com outras partes do CSS, tal como flexbox, possibilita a criação de layouts que até então eram impossíveis de serem criados com CSS. Tudo começa com a criação de um grid dentro de um grid contêiner.

Grid container flex_wrap

Para criar um grid-container temos de declarar no CSS a propriedade display com o valor grid ou inline-gride, assim: display: grid; para o elemento de marcação. Assim todos os elementos filhos diretos daquele contêiner se transformam em grid items.

Exemplo de sintaxe:

No exemplo aseguir vejamos como declar o layout grid ao elemento pai.

HTML

                    
                        <div class="l_container">
                            <div>item 1</div>
                            <div>item 2</div>
                            <div>item 3</div>
                            <div>item 4</div>
                            <div>item 5</div>
                        </div>

CSS

                    
                        .l_container {
                            display: grid;
                        }

Grid Tracks flex_wrap

Para definir linhas e colunas no layout grid temos de declarar as propriedades grid-template-columns: e grid-template-rows:, dessa forma criamos o grid tracks. Um grid track é o espaço entre duas linhas em um grid.

Exemplo de sintaxe:

No exemplo aseguir podemos ver como criar um grid track com a propriedade grid-template-columns. vamos criar três colunas fixas de 100px. os elementos filhos serão dispostos nessa grade, cada um em uma celula da grade.

HTML

                    
                        <div class="l_container">

                                <div>items 3</div>
                                <div>items 4</div>
                                <div>items 5</div>
                        </div>

CSS

                    
                        .l_container {
                                 display: grid;
                                 grid-template-columns: 100px 100px 100px;
                        }

Grid Auto Flow

O algoritimo de arrumação automatica é bastante útil para o posicionamento dos elementos itens no fluxo da grid container. Este algorítimo adicina mais trilhas de linhas ou colunas implícitas na grid quando adicionamos mais elementos itens além da quantidade de células existentes inicialmente. E existe uma propriedade que é usada para controlar este algorítimo, o grid-auto-flow. Esta propriedade nos permite manipular onde em qual trilha os novos elementos itens vão ficar posicionados. Para isso basta declará os valores:

  • row → (Padrão) posiciona os elementos itens nas linhas adicionadas automaticamente.
  • column → Posiciona os elementos itens nas colunas adicionadas automaticamente.

Vamos conferir um exemplo de sintaxe da propriedade grid-auto-flow:

Sintaxe

HTML

                    
                        <div class="l_container">
                                    <div>items 1</div>
                                    <div>items 2</div>
                                    <div>items 3</div>
                                    <div>items 4</div>
                                    <div>items 5</div>
                                    <div>items 6</div>
                        </div>

CSS

                    
                        .l_container {
                            display: grid;
                            grid-template-columns: 1fr 1fr;
                            grid-template-rows: 60px 60px;
                            grid-auto-flow: row;
                        }

A unidade fr

Nas propriedades de grid podem ser utilizadas quaisquer unidades de medida, mas para ajudar a criar layouts flexíveis utilizando o grid, foi criada uma nova medida: fração (fr). A unidade fr representa uma fração do espaço disponível no contêiner do grid.

Exemplo de sintaxe

O próximo exemplo vamos declarar grid-template-columns com a unidade de fração (fr):

Track listings com a notação repeat()

Podemos simplificar uma grid track muito grande com muitos valores repetidos usando a notação repeat(). A sintaxe é declarada da seguinte forma: "repeat", parenteses "()" e dentro colocamos a quatidade de colunas ou linhas iguais, em seguida uma virgula "," e por último a unidade de medida "fr, pixels, % etc..". Vejamos o exemplo aseguir:

CSS

                    
                        .l_container {
                                    display: grid;
                                    grid-template-columns: 1fr 1fr 1fr;
                        }

Com a notação repeat() simplificamos para:

CSS

                    
                        .l_container {
                                    display: grid;
                                    grid-template-columns: repeat(3,1fr);
                        }

Também, a notação repeat() pode ser usada para uma parte da track listing quando não são todos os valores de medidas iguais, mas apenas alguns valores iguais. Vejamos o exemplo:

CSS

                    
                        .l_container {
                                    display: grid;
                                    grid-template-columns:  20px repeat(6,     1fr) 20x;
                        }

A notação repeat() tem como parâmetro uma track listing, sendo assim você pode usá-lo para criar um padrão de repetição de tracks. No próximo exemplo vamos criar um grid de 10 tracks columns, uma trilha de 1fr seguida por uma trilha de 2fr, ambas repetidas 5 veses

CSS

                    
                        .l_container {
                                    display: grid;
                                    grid-template-columns: repeat(5, 1fr      2fr);
                        }

O grid implícito e explícito

Ao criar um grid layout apenas com uma trilha de colunas declarando grid-template-columns as colunas estarão no grid explícito. Mas as linhas que não foram definidas serão criadas pelo grid implícito automaticamente de acordo com a necessidade do conteúdo.

Grid Explícito é quando definimos explícitamente o número de colunas e linhas que criam um número de células que coincidem com o número exato de elementos do HTML. Vejamos o exemplo:

HTML

                    
                        <div class="l_container">
                                    <div>items 1</div>
                                    <div>items 2</div>
                                    <div>items 3</div>
                                    <div>items 4</div>
                                    <div>items 6</div>
                                    <div>items 7</div>
                        </div>

No exemplo temos seis elementos que coincidem com o número de celulas geradas.

CSS

                    
                        .l_container {
                                    display: grid;
                                    grid-template-columns: 100px 100px 100px;
                                    grid-template-rows: 60px 60px;
                        }

HTML

                    
                        <div class="l_container">
                                    <div>items 1</div>
                                    <div>items 2</div>
                                    <div>items 3</div>
                                    <div>items 4</div>

Os itens 7, 8, e 9 não fazem parte do grid explícito, mas sim parte do grid implícito. O navegador tentou adaptar os itens ao grid mas ele não iguala à altura das células explícitas, por isso declaramos a propriedade grid-auto-rows: com o valor da medida das células explícitas para que então o navegador iguale as células com a mesma altura das demais. Também podemos usar quando necessário a propriedade grid-auto-columns quando for o caso de o navegador não igular a largura de células implícitas à células explícitas.

                                    grid-auto-rows: 60px;

Dimensionamento de trilha e minmax() flex_wrap

A função de minmax() é tornar possível definir o tamanho de uma track da grid com um intervalo mínimo e máximo, para que o grid possa flexionar e se adaptar melhor à viewport de cada usuário da melhor maneira possível.

sintaxe

CSS

                        
                            .l_container {
                                        display: grid;
                                        grid-template-columns: minmax(100px, 200px) 1fr 1fr;
                                        grid-template-rows: 100px 100px;
                            }

Resultado:

item 1
item 2
item 3
item 4
item 5
item 6

minmax() é declarada como valor para as propriedades grid-template-columns e,ou, grid-template-rows. Nele inserimos uma medida mínima e medida máxima separados por virgula ",". Onde o valor min deve ser menor que o valor max, caso contrário o valor max será ingnorado pelo navegador.

Valores estáticos de comprimento

Existem duas maneiras de como podemos usar a função minmax() com valores estáticos de comprimento.

Primeiramente, podemos usar minmax() apenas em uma coluna do grid e defenir a largura das outras colunas com valores estáticos simples (pixel por exemplo), como mostrado abaixo.

CSS

                        
                            .l_container {
                                    display: grid;
                                    grid-template-columns: minmax(100px, 200px) 200px 200px;
                            }

Em segundo lugar, podemos definir a largura de mais de uma coluna do grid usando minmax().Os valores min e max são estáticos, portanto, por padrão o grid não é responsivo. Porém as próprias colunas serão flexíveis apenas entre o valor mínimo e o valor máximo. Elas crescem e encolhem simultaneamente à medida que mudamos o tamanho do layout.

CSS

                        
                            .l_container {
                                    display: grid;
                                    grid-template-columns:minmax(100px, 200px) 
minmax(100px, 200px) minmax(100px, 200px);
                            }

Podemos ainda usar a função repeat() junto como minmax(), o que pode sinplificar mais ainda o nosso código. vamos ver como fica a sintaxe:

CSS

                        
                            .l_container {
                                    display: grid;
                                    grid-template-columns: repeat(3,minmax(100px, 200px));
                            }

Valores dinâmicos de comprimento

Além dos valores estáticos, a função minmax() também aceita unidades de porcentagem (%) e a nova unidade de fração (fr) como valores. Ao usá-las, podemos obter grids personalizadas que são responsivas e alteram suas dimensões de acordo com o espaço disponível.

CSS

                        
                            .l_container {
                                    display: grid;
                                    grid-templates-columns: minmax(50%, 80%) 1fr 1fr;
                            }

Linhas virtuais de grid flex_wrap

Devemos observar que quando definimos um grid definimos as trilhas do grid. Já as linhas virtuais que contamos são as que estão disposta em parelelo às trilhas, e no entorno delas. Em uma trilha de três colunas e duas linhas, teremos "4" linhas virtuais no entorno das colunas, e "3" linhas virtuais entorno das linhas. Vejamos um exemplo na imagem a seguir:

Posicionado itens entre linhas virtuais

Entendendo sobre as linhas virtuais podemos então manipular os itens grids através das propriedades grid-column-start, grid-column-end e grid-row-start, grid-row-end para definir suas posições dentro do contêiner.

Para definirmos as posições dos itens devemos dizer em qual linha virtual ele vai iniciar e depois em qual ele vai terminar. Para essa definição temos de declarar as propriedades grid-column-start e grid-row-start para a linha virtual de inicio, e as propriedades grid-column-end e grid-row-end para a linha virtual de término. Vejamos o exemplo da sintaxe:

Nós podemos diminuir a quantidade de códigos CSS para atingir este mesmo resultado se usarmos as versões de propriedades shorthand grid-column e grid-row no lugar das propriedades variantes. Vejamos o exemplo abaixo:

CSS

                    
                        .l_container {
                                display: grid;
                                grid-template-columns: 1fr 1fr 1fr;
                                grid-template-rows: 80px 400px 60px 40px;
                        }
                        .l_header {
                                grid-column: 1/4;
                                grid-row: 1/2;         /* ou grid-row:2; */
                        }
                        .l_nav {
                                grid-column: 1/2;       /* ou grid-column: 2; */
                                grid-row: 2/3;         /* ou grid-row: 2; */
                        }
                        .l_main {
                                grid-column: 1/4;
                                grid-row: 2/3;         /* or grid-row:2; */  
                        }
                        .l_aside {
                                grid column: 1/4;
                                grid-row: 3/5;
                        }
                        .l_footer {
                                grid-column: 1/4;
                                grid-row: 4/5;          /* ou grid-row: 4; */
                        }

No shorthand de grid-column e grid-row o primeiro número corresponde a linha virtual de start, o simbolo de barra " / " indica até onde o elemento vai se estender, e o segundo número corresponde a linha virtual de end. No caso dos elementos que se estendem em apenas uma coluna ou linha grid podemos declarar somente o número do valor start, como está comentado nos códigos do exemplo /* ou grid-row: 4; */ , pois mesmo assim o os navegadores interpretarão os códigos corretamente.

Podemos ainda usar um "valor reservado -1" para que um elemento ocupe toda a extensão do contêiner. No exemplo vimos que o "Header" ocupa todas as três colunas do grid pois está declarado para ele o "grid-column: 1/4;". Porém se adicionarmos mais uma coluna para contêiner o "Header" não se estenderá, porque está delimitado para três colunas. Já se declararmos "grid-column: -1;" o "Header" se estenderá para mais colunas que adicionarmos.

Outro valor reservado para as propriedades grid é o span. span dertermina que o elemento item possa abranger uma quantidade de colunas ou linhas grid, apartir da linha virtual start, até a quantidade definida por ele (o span). vejamos o exemplo de sintaxe:

                    
                        .l_header {
                            grid-column:1/span 2;
                        }

Sobrepondo itens com z-index

Itens de uma mesma grid podem ocupar uma mesma célula sobrepondo um sobre o outro. Isso ocorre de acordo com o posicioncamento dos itens através das linhas virtuais. Além disso podemos também manipular a ordem desta sobreposição, ou seja qual item fica sobre o outro, usando a propriedade CSS z-index. Para isso devemos declará esta propriedade para os dois itens em questão, atribuindo os valores com números inteiros, e o item que está com o valor de z-index maior vai se sobrepor sobre o item de valor z-index menor.

No exemplo abaixo podemos ver como ocorre a sobrepodição dos itens de uma grid container.

Sintaxe

HTML

                    
                        <div class="l_container">
                                <div class="item__1">items 1</div>
                                <div class="item__2">items 2</div>
                                <div class="item__3">items 3</div>
                                <div class="item__4">items 4</div>
                                <div class="item__5">items 5</div>
                                <div class="item__6">items 6</div>
                        </div>

CSS

                    
                        .l_container {
                            display: gris;
                            grid-template-columns: repeat(3, 1fr);
                            grid-auto-rows: 100px;
                        }
                        .item__1 {
                            grid-colum:1/4;
                            grid-row:1/3;
                            z-index: 0;
                        }
                        .item__3 {
                            grid-colum:2/4;
                            grid-row:2/4
                            z-index: 1;
                        }

Posicionamento através de Grid Area flex_wrap

Quando criamos um "Grid container" e delimitamos as "tracks grid" automaticamente também criamos células (ou areas) onde são organizados os elementos itens. Uma célula de grid é a menor unidade em um grid container, conceitualmente seria como células de tabelas. E no CSS podemos nomear cada célula da grid usando a propriedade grid-area no seletor CSS do elemento item.

Já no seletor do elemento contêiner podemos adicionar a propriedade grid-template-areas. Essa propriedade nos permite posicionar cada elemento item nas "tracks grids" da maneira que queremos. Estas propriedades trabalham coletivamente. No grid-template-areas atribuimos o nome das grid-areas de maneira que os nomes correspondam a posição de cada célula da "grid", podendo também repetir os nomes quando uma grid-area ocupa mais de uma célula.

Vamos ver uma exemplo de sintaxe desta técnica de grid:

HTML

                    
                        <div class="l_container">
                            <div class="l_header">Header</div>
                            <div class="l_nav">Nav</div>
                            <div class="l_main">Main</div>
                            <div class="l_aside">Aside</div>
                            <div class="l_footer">Footer</div>
                    </div>

CSS

                    
                        .l_container {
                            display: grid;
                            grid-template-columns: 1fr 3fr 1fr;
                            grid-template-rows: 60px 300px 50px 40px;
                            grid-template-areas: 
                                "header  header  header"
                                "nav  main  main"
                                "aside  aside  aside"
                                "footer  footer  foter"
                        }
                        .l_header {
                                grid-area: header;
                        }
                        .l_nav {
                                grid-area: nav;
                        }
                        .l_main {
                                grid-area: main;
                        }
                        .l_aside {
                                grid-area: aside;
                        }
                        .l_footer {
                                grid-area: footer;
                        }

No exemplo os nomes estão organizados para representar as células geradas pelas trilhas de colunas e de linhas grid do elemento contêiner (sendo esta uma das boas práticas para essa técnica). Os valores para grid-template-areas são declarado entre aspas duplas ("") ou aspas simples (''), e os nomes do grid-area não precisam de aspas.

Guntter: Espaçamento de grid-area

Guntters são espaçamentos que separam as células da 'grid' no eixos X e no eixo Y. Estes espaços são criados usando as propriedades grid-column-gap e grid-row-gap. Os valores para estas propriedades podem ser declaradas em "px" ou "em". Também existe a forma reduzida com a propriedade grid-gap, onde a sequência dos valores fica assim: o primeiro valor representa o eixo X e o segundo valor representa o eixo Y. Os espaçamentos são adicionados apenas entre as células da "grid", não adiciona espaço para as margens do grid container.

  • grid-column-gap: → Adiciona espaço entre as colunas , no eixo Y.
  • grid-row-gap: → Adiciona espaço entre as linhas, no eixo X.
  • grid-gap: → É um shorthand para adicionar espaço para os dois eixos.

HTML

                    
                        <div class="l_container">
                        <div>item 1</div>
                        <div>item 2</div>
                        <div>item 3</div>
                        <div>item 4</div>
                        <div>item 5</div>
                        <div>item 6</div>
                    </div>

CSS

                    
                        .l_containar {
                            grid-template-columns: repeat(3, 1fr);
                            grid-template-rows: 60px 60px;
                            grid-column-gap: 10px;
                            grid-row-gap: 1.2em;
                        }

Aninhamento de grids

Um item de uma grid também pode se tornar um grid container. Isso acontece porque se o elemento filho tiver mais elementos contidos dentro dele, eles não herdam atributos e nem participam do grid layout, desta forma são exibidos no fluxo normal do documento. Então podemos aninhar grids containers adicionando display: grid para o elemento filho, nos permetindo assim estilizar outra grid container dentro de uma já existente.

Vamos ver uma exemplo de sintaxe de aninhamento de grids container.

Sintaxe

HTML

                    
                        <div class="l_container">
                                <div class="l_items">
                                        <div>item a</div>
                                        <div>item b</div>
                                        <div>item c</div>
                                </div>
                                <div>item 2</div>
