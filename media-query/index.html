O que é Media Query

A Media Query permite que sejam aplicadas diversas formatações para atender diferentes dispositivos e tamanhos de telas. E com isso podemos proporcionar que os usuários tenha uma boa experiência ao navegar através das páginas de nosso site ou aplicação. Ou podemos até mesmo formatar as páginas do website para a opção de impremir.

Na prática, Media Qurey permitem criar layauts que moldam a imformação do website de forma que fique apresentável de acordo com o tamanho da viewport do dispositivo do usuário

Sintaxe

As Media Query derivam da Media Type, e, para que a aplicação interprete corretamente as especificaçôes CSS3 disponibilizadas, temos as chamadas Media Features, que retornam verdadeiro ou falso. Para que o retorno seja verdadeiro, é preciso que o retorno da Media Type, que é especificado na Media Querey, esteja de acordo com o tipo de documento do dispositivo.

Técnicamente Media Query são Media Type que fazem um estilo valer a denpender de derterminadas condições especidicadas. Inicialmente uma Media Query era declarada como link no head do documento HTML como um arquivo de estilização específico para uma determinada condição. Exemplo:

            
                <link rel="stylesheet" type="text/css" href="print.css" media="print">

Com o passar do tempo as Media Query poderam ser introduzidas dentro de um arquivo CSS declarando o comando@media mais a "condição" ou Media type(all, screen, print e speech) e em seguida dentro de {...} especificando-se a estilização desejada. Se nenhum media type for especificado, por padrão estará declarado implícitamente a condição "all".

  • screen: Todas as telas de dispositivos que não são print ou speech;
  • print: Direciona o arquivo para dispositivos de impressão
  • speech: Direciona o arquivo para sintetizadores de fala:
  • all: Padrão. Direciona o arquivo para todos os media types anteriores

Exemplo

            
                @media screen and (min-width: 768px) {
                    .content{
                        ...regras CSS
                    }
                    header{
                        ...regras CSS
                    }
                }

Alem das Media Type também existem disponíveis as Media Features que descrevem caracteristicas específicas de user agents, dispositivos ou do próprio ambiente. aluguns exemplos são: "color", " pointer", "orientetion". Media features também servem para testar se uma condição é verdade, mas, ao invés de testarem meios, testam características. A lista de Media Feature disponível é extensa. A seguir teremos alguns exemplos.

  • width, max-width e min-width
  • height, max-height e min-height
  • orientation
  • pixel-density
  • aspect-ratio
  • prefers-color-scheme

Operadores Lógicos

Os operadores lógicos podem ser usados para compor uma consulta de mídia complexa: not, end e only. Podemos também combinar várias consultas de mídia em uma única regra, separando-as com vírgulas. Uma consulta de mídia calcula true quando o tipo de mídia (se especificado) correspode ao dispositivo no qual um documento está sendo exibido e todos as expressões de recurso de mídia são computadas como verdadeiras. As consultas envolvendo tipos de mídia desconhecidos são sempre false.

  • and: Operador capaz de combinar varios Media Features em uma única regra de Media Query.
  • not: Operador capaz de negar uma Media Feature basicamente invertendo seu significado normal.
  • only: Operador que impede que navegadores mais antigos apliquem os estilos da regra da Media Query.

Exemplo

            
                @media (min-width: 600px) and (orientation: landscape) {
                    body {
                        background-color: red;
                    }
                }

Segmentação de Media type

Os Media Type descrevem a categoria geral de um determinado dispositivo. Embora os sites sejam geralmente projetados com telas em mente, você pode quere criar estilos que visam dispositivos especiais, como impressoras ou leitores de tela baseados em áudio. Sabendo disso, podemos também segmentar vários dispositivos na mesma regra de Media Query. como por exemplo:

            
                @media screen, print {
                    width: 100%;
                }

Segmentação de Media Feature

Como as Media Features descrevem características específica de um determinado user agent, dispositivo de saída ou ambiente (como widescreen, computadores que usam mouse ou dispositivos usados em condições de pouca luz), podemos combina-las juntos ao mesmo tempo na mesma regra de Media Query usando os operadores de lógica.

Exemplo

            
                @media (orientation: portrait), (min-width: 3rem) and (max-aspect-ratio: 2/1) {
                    margin: 0 auto;
                }

Breakpoints

Breakpointssão, talvez, o termo mais comum usados quando falamos de Media Query. Um breakpoint é uma chave usada para determinar o momento de pausa para mudar o layout, e adaptá-lo às novas regras dentro de uma media query. Vamos à um exemplo:

            
                @media (max-width: 480px) {
                    .text {
                        font-size: 1em;
                    }
                }

Aqui, o breakpoint é 480px determinado por uma media feature. Agora, a media query sabe quando definir ou sobrescrever a nova classe. Basicamente, se a largura de um dispositivo for inferior a 480px, a classe text será aplicada. Do contrário, isso não acontece.

Breakpoints comuns

Hoje em dia, existem milhares de dispositivos no mercado com grande variedade de tamanhos de tela que acessam os navegadores de internet. Então não podemos definir breakpoints fixos para cada um deles. Sendo assim, o que os desenvolvedores de websites utilizam no seu dia a dia são as resoluções mais comuns e que abrangem o maior número de tamanhos de telas possível. A seguir esta uma lista com essas resoluções.

  • 360px até 480px: dispositivos móveis
  • 481px até 768px: iPads e tablets
  • 769px até 1024px: telas pequenas, laptops
  • 1025px até 1200px: desktops, telas grandes
  • 1201px e acima: telas muito grande e tvs.

Mobile First

Mobile First é um conceito aplicado em projetos web onde o foco inicial da arquitetura e desenvonlvimento é direcionado aos dispositivos móveis e em seguida com foco secundário é direcionado para as demais telas como de tablet, desktop, tvs e etc...
